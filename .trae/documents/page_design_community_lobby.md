# 大厅/社区首页（Desktop-first）页面设计说明

## 1) Layout
- 采用「顶部统计 + 中部列表 + 底部悬浮入口」的三段式结构。
- 主容器：`max-width: 1200px; margin: 0 auto; padding: 24px;`。
- 顶部统计：CSS Grid（`grid-template-columns: repeat(3, 1fr)`），卡片间距 16px。
- 房间列表：垂直堆叠卡片（Flex column，gap 12px），支持分页/无限滚动占位骨架。
- 底部悬浮：`position: fixed; right: 24px; bottom: 24px;` 按钮组；滚动时始终可见。
- 响应式（桌面优先）：
  - ≥1024px：统计 3 列；列表与抽屉并行体验最佳。
  - 768–1023px：统计改 2 列；按钮组保持右下。
  - <768px：统计单列；房间详情用全屏弹窗替代侧边抽屉。

## 2) Meta Information
- Title: 社区大厅
- Description: 浏览公开房间、查看实时统计、申请加入或通过邀请码进入。
- Open Graph:
  - og:title = 社区大厅
  - og:description = 公开房间列表与实时社区概览

## 3) Global Styles（设计令牌）
- 背景色：`--bg: #0B1020`（深色）或 `#F7F8FA`（浅色，可二选一项目主题）
- 卡片：`--card: #111827` / `#FFFFFF`；圆角 12；阴影 1 级
- 主题色：`--primary: #4F46E5`
- 强调色：成功 `#16A34A`、警告 `#F59E0B`、危险 `#DC2626`
- 字体：标题 20/24/28，正文 14/16，辅助文字 12
- 按钮：
  - Primary：实心主题色；hover 提亮；disabled 降低不透明度
  - Secondary：描边/浅底；hover 背景轻微加深
- 链接：主题色；hover 下划线

## 4) Page Structure
1. 顶部：页面标题 + 实时统计卡片区
2. 中部：公开房间列表（含加载/空态/错误态）
3. 覆层层：房间详情抽屉/弹窗、创建房间弹窗、邀请码加入弹窗、登录引导弹窗
4. 底部：悬浮操作按钮组

## 5) Sections & Components

### 5.1 顶部栏（Header）
- 左侧：页面标题「社区大厅」+ 小号副标题（例如“实时概览”）。
- 右侧：
  - 主按钮「快速开始」（Primary）：一键进入推荐房间或一键创建默认房间（见 PRD“快速开始流”）。
  - 登录态信息（头像/昵称）与“登录”入口（未登录）。
- 交互：快速开始在 loading 时展示按钮内 spinner；失败 toast 给出原因（如“暂无可加入房间，建议创建”）。

### 5.2 实时统计（Stats Strip）
- 3 张统计卡（可横向滚动以适配窄屏）：
  1) 在线人数（基于 last_seen_at 口径）
  2) 公开房间数
  3) 待处理申请数（与你相关：房主/申请人各自口径）
- 卡片内容：数值（大号）、label（小号）、“更新时间/正在刷新”状态。
- 交互：刷新中显示骨架或轻量 loading indicator；错误时显示“重试”。

### 5.3 公开房间列表（Public Rooms List）
- 列表项（Room Card）字段：
  - 主信息：房间名、简介（2 行截断）
  - 统计：成员数/上限、房主名、最近活跃时间（或创建时间）
  - 状态徽标（与房间状态联动）：
    - 已满：badge「已满」（优先级最高）
    - 待审核：badge「待审核」（展示剩余有效时间，例如“还剩 12h”）
    - 冷却中：badge「冷却中」（展示倒计时，例如“15min”）
    - 已加入：badge「已加入」
    - 已拒绝：badge「已拒绝」（若未进入冷却则可再次申请）
    - 申请已过期：badge「已过期」（提示可重新申请）
  - CTA：
    - 默认按钮「查看」；在 Drawer 内完成加入/申请
    - 若已满/冷却中：按钮禁用并显示原因 tooltip
- 空态：文案“暂无公开房间”；提供按钮“创建房间”与“快速开始”。
- 错误态：提示“加载失败”；提供“重试”。

### 5.4 房间详情抽屉/弹窗（Room Detail Drawer/Modal）
- 展示：房间名、简介、成员数/上限、房主、加入规则提示（默认“需审核加入” + 明确人数上限规则）。

**Drawer 交互规范（Desktop-first）**
- 触发：点击房间卡片「查看/加入」或快速开始自动打开。
- 形态：桌面右侧滑出 Drawer（建议宽 440–520px），带遮罩；移动端降级为全屏 Modal。
- 关闭：右上角关闭按钮、按 `Esc`、点击遮罩；存在未提交输入时二次确认（可选）。
- 焦点与可访问性：打开时 focus 到 Drawer 标题；Tab 在 Drawer 内循环；关闭后焦点回到触发按钮。
- 滚动：Drawer 内容区独立滚动；标题区固定（sticky）以便随时关闭。

**你的状态区（内容与状态联动）**
- 已满：显示“人数已满”，所有加入/申请 CTA 禁用。
- 未登录：提示登录；按钮“去登录”（携带 redirect+intent+roomId）。
- 可申请：附言输入（可选）+ 按钮“提交申请”。
- 待审核（未过期）：展示“等待房主处理” + 过期倒计时；提供“撤回申请”（若产品已支持）否则仅展示。
- 申请已过期：展示“申请已过期，请重新申请” + 重新申请按钮。
- 冷却中：展示冷却倒计时，提交按钮禁用。
- 已加入：展示“你已加入”，入口可替换为“进入房间”（若产品已有房间内页）。
- 已拒绝：展示拒绝原因（如有）与可再次申请条件。

**房主视角（仅当你为 owner）**
- 申请列表：仅显示 pending（列表项含申请人、附言、时间、剩余有效期）。
- 操作：同意/拒绝（拒绝可填备注）；同意前提示“若房间已满将无法同意”。

### 5.5 创建房间弹窗（Create Room Modal）
- 表单字段：房间名（必填）、简介（可选）、人数上限（必填，默认 50）。
- 提交：创建成功后关闭弹窗，列表顶部出现新房间；你成为房主。

### 5.6 邀请码加入弹窗（Invite Code Modal）
- 输入：邀请码（必填，支持粘贴）。
- 校验结果：
  - 成功：提示“加入成功”，关闭并刷新你的状态。
  - 失败：明确原因（过期/次数用尽/不存在）。

### 5.7 底部悬浮操作（Floating Actions）
- 组件：一个主 FAB + 两个子按钮，或并排双按钮。
  - 「创建房间」Primary
  - 「邀请码加入」Secondary
- 与“快速开始”的关系：
  - 快速开始放在 Header 作为强引导入口；悬浮入口保留为“明确操作”（创建/邀请码）。
- 未登录：点击任一按钮先弹出登录引导（含“去登录/取消”，并携带 redirect+intent）。
