# 谁是卧底 Web 项目执行说明

## 项目描述
构建一个基于 Next.js 16 的“谁是卧底”网页应用，支持快速开局、多人线下同屏查看身份词，并具备以下能力：

- 输入总人数并自动计算合理卧底人数（允许手动微调）
- 支持预设类别词库：游戏、化妆品、生活、网络词汇
- 支持用户自定义类别词（可保存到本地浏览器）
- 可调用 Grok API 自动生成某类别的词对，扩充词库
- 页面风格简洁、美观、易操作，适配移动端和桌面端
- 可直接部署到 Vercel

## 技术方案

- 前端：Next.js App Router + React 19 + TypeScript
- 样式：全局 CSS 变量 + 轻量动画（不引入额外 UI 框架）
- 服务端接口：Next.js Route Handler（`/api/grok/words`）
- 外部能力：Grok API（通过服务端调用，密钥走环境变量）
- 本地数据：浏览器 localStorage 存储自定义词条

## 执行步骤

1. 需求落地与界面骨架
- 设计单页流程：参数设置 -> 生成身份 -> 按玩家逐个查看
- 实现信息层级：人数、卧底人数、词库类别、自定义词条入口
- 验收：可以在一个页面内完成从配置到开始游戏

2. 游戏核心逻辑
- 根据总人数计算默认卧底人数（分段规则）
- 生成玩家身份列表（卧底/平民）并打乱顺序
- 按玩家序号逐个揭示词语，避免一次性泄露
- 验收：同一局内词语一致、卧底数量准确、顺序随机

3. 词库系统
- 内置预设类别与词对数据
- 支持用户自定义类别名和多组词对（平民词/卧底词）
- 合并预设与自定义词库并统一供游戏抽取
- 验收：自定义词在刷新后仍存在，且可用于开局

4. Grok API 集成
- 新增 `/api/grok/words` 接口，输入类别与数量返回词对
- 实现服务端提示词约束和返回 JSON 解析与兜底
- 前端提供“一键生成并加入自定义词库”操作
- 验收：配置 `GROK_API_KEY` 后可生成词条并可直接开局

5. 视觉与交互优化
- 建立统一设计变量（颜色、圆角、阴影、间距）
- 增加少量动效（卡片淡入、按钮反馈、步骤切换）
- 响应式布局优化，确保手机可单手操作
- 验收：界面清晰、操作路径短、观感简洁现代

6. 部署与文档
- 更新 README：本地运行、环境变量、Vercel 部署步骤
- 提供 `.env.local` 示例项说明
- 验收：按文档可一键部署并在生产可用

7. 质量检查
- 运行 `npm run lint` 并修复错误
- 手动验证关键流程：开局、揭示、重开、词库增删
- 验收：无阻塞错误，主要流程可稳定运行

## 环境变量计划

- `GROK_API_KEY`：Grok API 密钥（服务端使用）
- `GROK_API_URL`：可选，自定义 API 地址（默认官方地址）
- `GROK_MODEL`：可选，默认 `grok-2-latest`
